# 条件节点 (Condition)

*Condition*（条件）节点被设计为二元门控。

它们将播放时内存（控制台或运行时）中[变量](../getting-started/variables-logic)的当前值与以下内容进行比较：

+ 一个已知的静态值
+ 另一个变量的当前值
+ 或它们自己的初始值

然后根据特定标准判断这些左右手边的值是否匹配。

所有这些参数都可以由用户编辑。

每种变量类型都有其自己支持的标准集，并且只能比较相同类型的变量。

如果匹配标准通过，则播放正向或 `True` 出站槽，否则播放负向或 `False` 出站槽。

（伪）公式显示在每个节点的网格视图上，以表示条件语句。

### 约定

+ 按照约定，被跳过的 Condition 节点会首先尝试播放其 `False` 槽。如果该槽没有出站连接，则播放 `True` 槽。

+ Condition 是自动节点：它不等待用户交互。

### 使用

Condition 节点的常见用途是根据设计师期望的标准和可能在播放时间内变化的状态，引导玩家通过分支故事。换句话说，根据可变的游戏状态控制事件。

例如，您可以创建一个 `player_rhetoric`（数字）变量来跟踪玩家在外交语言方面获得的分数。然后放置一个 *Condition* 门控，根据他们的分数是高于还是低于某个特定值，或您跟踪的另一个分数（如 `player_honesty`），分支到两个不同的子剧情。

> 快速提示！  
> 您可以使用 Condition 节点在内部对作品进行版本控制。  
> 例如，如果您的叙事包含成人内容或暴力内容，您可以将这些内容放在条件门控后面，只有当观众达到适当年龄并愿意时才打开。
