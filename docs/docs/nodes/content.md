# 内容节点 (Content)

*Content*（内容）节点非常适合在叙事中显示大段文本。

它们接受一组*全部可选*的参数，包括：

+ 标题 (Title)
+ 内容正文 (Content body)
+ 摘要长度 (Brief length)
+ 自动播放 (Auto-play)
+ 清除 (Clear)

`title`（标题）和 `content`（内容）正文参数预期显示给玩家。根据运行时实现，它们可能有不同的样式和视图。

另一个参数 `brief`（摘要）长度更多是 Arrow 编辑器的功能，它指示从 `content` 正文开头起多少个字符将在节点的网格视图上预览。`-1` 表示全部文本。
> 自定义运行时可以将此参数用于创意目的，例如折叠内容正文。

当可选的 `auto`（自动）播放参数为 `true`（或未设置且默认为 `true`）时，内容将自动播放下一个节点，否则将等待用户交互才能继续。

Content 节点还可以选择在显示给玩家之前 `clear`（清除）视图。它特别针对文本墙显示，如果这是您的运行时显示内容的方式。

### 约定

+ 被跳过的 Content 节点不执行任何操作（不打印文本，不清除页面），并自动播放下一个节点（无论 `auto` 和 `clear` 参数如何）。

+ 标题和内容正文视图都预期支持最小的 *BBCode* 样式。

+ 标题和内容正文参数都支持[变量](../getting-started/variables-logic)和[角色](../getting-started/characters)标签暴露。

### 使用

Content 节点简单、通用且直接。它们可用于向玩家提供信息、场景描述或任何其他没有特定节点类型的内容。

> 快速提示！  
> 您可以使用 `auto`（自动）播放参数通过条件混合多个原子内容片段来创建相对较大或动态的内容正文。
>> 让最后一部分手动播放可能也是个好主意，给用户时间阅读您的文本。
