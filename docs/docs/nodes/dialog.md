# 对话节点 (Dialog)

*Dialog*（对话）节点是*部分对话*。

换句话说，每个 Dialog 节点传达对话的一方，尽管名称可能有点误导。

这种设计加上可播放选项，使 Arrow 对话系统更加灵活。

Dialog 节点属于[角色](../getting-started/characters)。它们的 `character` 字段也可以是匿名的（`-1`），尽管不推荐这样做。

这些节点接受一行或多行。如果可选的 `playable` 参数为 `true`（或未设置且默认为 `true`），节点将等待玩家选择一行，否则将随机播放一行。

### 约定

+ 如果被跳过，Dialog 节点播放第一个连接的出站槽，如果没有连接则播放第一个。

+ Dialog 行预期支持最小的 *BBCode* 样式。

+ Dialog 行支持[变量](../getting-started/variables-logic)和[角色](../getting-started/characters)标签暴露。

+ Dialog 节点（除非被跳过）将继续任何由用户交互或随机播放的断开连接的行。它们将其视为设计师/开发者的故意选择。自然地，断开连接的出站槽将结束分支（即 EOL）。

+ 不允许空白行（即长度为零的行）。

### 使用

每行都有一个出站槽，意味着您可以根据用户选择的对话行来分支您的故事，或创建非常详细的（乒乓式）讨论。

> 要创建完整的（多边）对话，您可以链接 Dialog（和/或 [Monolog](./monolog)）节点。

> 如果某些或所有 Dialog 行应该转发到单个下一个节点，您可以使用 [Hub](./hub)。

您可以使任何 Dialog 节点实例可播放，无论角色是 NPC 还是可玩角色。将此参数视为您是否希望由最终用户选择一行，还是由机器随机选择。
