# 入口节点 (Entry)

*Entry*（入口）不是叙事（文本）节点类型，而是技术上重要的导航节点。

Arrow 控制台和任何其他运行时实现都需要知道每个*场景*（或*宏*）从哪里开始。它们还需要知道哪个节点是整个项目的开始。

从技术上讲，您可以从任何节点开始；但专门设计的节点 Entry 可以提供优势，包括连续性安全所需的编辑器检查。

它们还有一个 `plaque` 参数，有助于更容易地识别它们。

### 约定

+ Entry 是自动节点：它不等待用户交互。

+ 无论是否被跳过，Entry 节点始终播放其唯一的出站槽。

+ 每个场景/宏只能有一个活动的 Entry，每个项目文档也只能有一个。一个实例也可以同时为两者都是活动的。

### 使用

虽然每个项目、场景或宏只需要并接受一个活动的 Entry 节点，但您可以创建任意数量的它们。

额外的（停用的）Entry 节点可以作为干净的起点，以防您在一个场景中有多个几乎独立的剧情线。它们也可以作为一个或多个 [Jump](./jump) 节点的焦点目标。
