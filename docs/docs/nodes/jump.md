# 跳转节点 (Jump)

Jump（跳转）是另一种技术上重要的导航节点类型。

Arrow 的编辑器在设计时考虑到了作者和艺术家，因此广泛采用图形（节点和链接）或无代码开发方法。节点（最常见的资源）主要在网格上组织并使用图形链接连接，创建定义这些构建块将按什么顺序播放的系列。

还应该有一种方法来连接不在同一网格上的节点（即不同场景上的节点），或者在同一网格上但相距很远的节点（正常的图形链接看起来不太好）。

这就是 *Jump* 节点类型的用武之地。

Jump 节点接受另一个节点的*名称*（用户可编辑的 UID），在内部找到相应的底层不可变 UID，并将其保存为 `target`，因此即使目标节点稍后重命名，也能保持有效的工作链接。

Jump 实例在其网格节点视图上显示其目标 `name` 和 `reason` 标签，有助于更好地可视化每个 Jump 及其目的。

当 Jump 节点被播放时，它立即播放其目标节点向前。

### 约定

+ Jump 是自动节点：它不等待用户交互。

+ 被跳过和无效或未设置的 Jump 节点不会播放任何内容向前（即 EOL）。

+ 跳转到另一个 Jump 被认为是*不安全操作*，因此是*意外行为*，可能会或可能不会被处理。

+ 跳转到属于宏的节点将导致该宏被视为普通场景。
    > 如果您打算从头到尾播放宏，请尝试推荐的方式：[宏使用 (Macro-Use)](./macro-use)。

### 使用

在 Jump 检查器的目标字段中输入另一个节点的 `name` 就是您需要做的全部。如果名称完全匹配，任务就完成了。对于部分名称，检查器会进行搜索并提供其名称中包含该子字符串的节点列表。然后您可以从列表中选择一个。

> 如果网格节点显示目标没有变化或检查器字段重置，则数据要么没有（自动）更新，要么输入的名称无效。

为每个 Jump 分配原因是可选的，但推荐这样做，特别是如果目标节点的名称不够有意义。

在网格 Jump 节点上 `Alt + 双击` 将更改网格视图以聚焦于目标节点。
