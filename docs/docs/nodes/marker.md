# 标记节点 (Marker)

*Marker*（标记）节点被设计为在编辑器网格和控制台上提供通用视觉标签。

它们有一个简单的文本 `label`（标签）和一个 `color`（颜色），使它们在网格及其小地图上明显可见。

### 约定

+ Marker 是自动节点：它不等待用户交互。

+ 无论是否被跳过，Marker 节点始终播放其唯一的出站槽；但如果被跳过，它们可能会或可能不会显示给玩家，取决于运行时。

### 使用

Marker 节点是很好的注释手段。

您可以使用它们标记需要稍后关注的地方，标记为 `WIP`（即工作进行中）、`TODO`、`NOTE` 等，带有独特的（约定的）颜色。您还可以向它们添加额外的开发者*节点笔记*，分享您的想法或跟踪要完成的任务。

> 节点笔记将在许多导出格式中被清除，但会保留在保存的项目中。

我们的小 Marker 朋友如何使用及其行为，高度取决于开发者的工作流程以及它们在运行时代码中如何实现。

Arrow 的控制台和[官方 HTML-JS 运行时][runtime-html-js]（主要被设计为播放测试、审查和调试工具）倾向于向玩家显示活动的标记。

> 如果标记仅用于网格上的注释，您可以跳过任何 Marker 以避免在播放时显示它们。

自定义工作流程（和运行时）可能决定使用标记来发出事件信号。这些事件可能被运行时钩子捕获和处理。Marker 的 `label` 参数也可用于定义应调用哪个函数和/或应传递什么参数。这是一种快速且简单的方式来原型化和扩展 Arrow，而无需花时间创建漂亮的自定义节点类型。
