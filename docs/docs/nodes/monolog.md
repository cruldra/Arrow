# 独白节点 (Monolog)

*Monolog*（独白）节点用于显示长篇演讲、思想和长篇部分对话。

与 [Dialog](./dialog) 节点相比，Monolog 不提供分支。另一方面，它们提供了类似于 [Content](./content) 节点的文本高级优势。

> 如果您有由角色说出的短行并导致不同分支，[Dialog](./dialog) 就是您想要的。

Monolog 节点属于[角色](../getting-started/characters)。必需的 `character` 也可以是匿名的（`-1`），尽管不推荐这样做。

第二个必需参数是 `monolog`，这是角色的表达，预期显示给玩家。

其他可选参数包括 `brief`（摘要）长度、`auto`（自动）播放和 `clear`（清除）。

数字（整数）参数 `brief` 长度更多是 Arrow 编辑器的功能，它指示从 `monolog` 开头起多少个字符将在节点的网格视图上预览。`-1` 表示全部文本。
> 自定义运行时可以将此参数用于创意目的，例如折叠长文本。

当可选的 `auto`（自动）播放参数为 `true`（或未设置且默认为 `true`）时，Monolog 节点立即播放其下一个节点，否则等待用户交互才能继续。

Monolog 节点还可以选择在显示给玩家之前 `clear`（清除）视图。它特别针对文本墙显示，如果这是您的运行时显示内容的方式。

### 约定

+ 被跳过的 Monolog 节点不执行任何操作（不打印文本，不清除页面），并自动播放下一个节点（无论 `auto` 和 `clear` 参数如何）。

+ Monolog 文本视图预期支持最小的 *BBCode* 样式。

+ Monolog 文本支持[变量](../getting-started/variables-logic)和[角色](../getting-started/characters)标签暴露。

### 使用

我们可以使用 Monolog 来传达由角色表达的长篇内容。它可以是对话一方说出的一长串话、角色的演讲、角色的思想等。

> 要创建完整的（多边）对话，我们可以链接 Monolog（和/或 Dialog）节点。

您可以使任何 Monolog 节点实例 `auto`（自动）播放，无论角色是 NPC 还是可玩角色。将此参数视为您是否想等待用户交互或只是继续。

> 快速提示！  
> 您可以使用 `auto`（自动）播放参数通过条件混合多个原子 Monolog 来创建相对较大或动态的独白。
>> 让最后一部分手动播放可能也是个好主意，给用户时间阅读您的文本。
