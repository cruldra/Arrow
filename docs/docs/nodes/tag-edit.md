# 标签编辑节点 (Tag-Edit)

*Tag-Edit*（标签编辑）节点用于以多种方式创建、删除或操作角色标签。

每个 Tag-Edit 节点接受一个标签键和一个标签值，并使用它们对其目标 `character`（角色）的标签集合应用以下编辑之一：

+ **Inset（插入）**：仅当键不存在时添加 `key:value` 标签。
+ **Reset（重置）**：仅当键存在时重置标签的值。
+ **Overset（覆盖）**：无论键是否存在，都覆盖或添加 `key:value` 标签。
+ **Outset（移除）**：如果标签的键和值都与编辑中的匹配，则从集合中移除该标签。
+ **Unset（取消设置）**：如果标签的键与编辑中的匹配，则移除该标签。

### 约定

+ Tag-Edit 是自动节点：它不等待用户交互。

+ 被跳过的 Tag-Edit 播放其唯一的出站槽向前，而不对其目标角色的当前标签集合应用任何更改。

+ 遵循角色检查器，标签键和值应为字符串；但只有值可以为空。

### 使用

角色标签是我们唯一可以在运行时创建的数据类型。
> 其他资源如[变量](../getting-started/variables-logic)，虽然在运行时可变，但应该从一开始就存在并有初始值。
这种动态性允许我们将它们用于临时或未定义的状态，或者当我们需要按需创建和管理相对大量的状态时。

标签的一个非常常见且强大的用途是事件管理。Tag-Edit 节点可以设置或取消设置几乎无限数量的事件标志，通过不同的分支；然后在适当的时候 [Tag-Pass](./tag-pass) 节点可以检查是否所有或任何通过特定分支所需的标志都已设置。

例如，您可能打算允许玩家通过血腥之门进入最终地牢，条件是英雄在一个分支中杀死了特定的怪物并在另一个分支中学会了魔法词。您可以使用 Tag-Edit 节点分别在上述分支中设置带有 `slew_big_bad_monster` 和 `knows_magic_word` 键的标签；然后在各处使用 Tag-Pass 节点检查英雄是否拥有所有力量（标签键）以通过。
> 当然，您可以使用[变量](../getting-started/variables-logic)配合[变量更新](./variable-update)和几个[条件](./condition)节点来实现类似的结果，但具有不同的人体工程学。两种方式都有各自的优缺点。这完全是您的选择，使用任何您认为适合您工作流程的方法。

> 快速提示！  
> 如果您打算一次又一次地检查事件，使用复杂的（多通道）逻辑，请考虑创建宏并利用 [Macro-Use](./macro-use) 节点。

> 查看[角色](../getting-started/characters)文档，了解更多关于标签及其其他令人兴奋的用途的信息。
